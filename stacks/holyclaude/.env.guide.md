# HolyClaude `.env` Guide (Secret-Safe)

**File:** `holyclaude/.env`
**Purpose:** Local-only environment values for the HolyClaude stack.

This guide intentionally uses placeholders only. Do not paste real tokens, keys, or webhook URLs into this document.

## Source of truth

- Runtime variables expected by compose are defined in:
  - `holyclaude/compose.yaml`
  - `holyclaude/.env.example`
- This guide explains how to fill `.env` safely.

## Required variables

```dotenv
PUID=0
PGID=0
TZ=America/New_York
WORKSPACE_PATH=/volume1/​docker/dockge/stacks
```

Notes:

- **NAS default:** `PUID`/`PGID` = **0** (root) per `HIVE_OBJECTIVE.md`. Use non-zero only on local Linux dev if you need host file ownership to match a normal user.
- `WORKSPACE_PATH` should point to the **stack root** so `/workspace/.claude-flow` is available inside the container.
- On non-NAS local runs, `/Volumes/docker/dockge/stacks` is acceptable.

## Notifications (Discord first)

```dotenv
DISCORD_WEBHOOK_URL=<discord_webhook_url>
NOTIFY_DISCORD=discord://<webhook_id>/<webhook_token>
```

Enable notifications from inside container:

```bash
touch /home/claude/.claude/notify-on
```

Disable:

```bash
rm /home/claude/.claude/notify-on
```

## Optional provider variables

```dotenv
ANTHROPIC_API_KEY=
ANTHROPIC_AUTH_TOKEN=
ANTHROPIC_BASE_URL=
CLAUDE_CODE_USE_BEDROCK=
CLAUDE_CODE_USE_VERTEX=

OPENAI_API_KEY=
```

Set only what you use; leave the rest blank.

## Security rules

- Keep `holyclaude/.env` local-only.
- Never commit populated `.env`.
- Commit placeholders in `.env.example` only.
- Rotate any key immediately if exposed.

## Verification commands

```bash
cd /volume1/​docker/dockge​/stacks/holyclaude
docker compose --env-file .env config
docker compose --env-file .env up -d
docker exec holyclaude sh -lc 'ls /workspace/.claude-flow'
docker exec holyclaude sh -lc 'echo ok > /workspace/.claude-flow/holyclaude-write-test.txt && rm /workspace/.claude-flow/holyclaude-write-test.txt'
```

## Persistence reminders
